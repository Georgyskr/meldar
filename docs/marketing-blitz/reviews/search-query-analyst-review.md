## 2. Negative Keyword Architecture -- Day 1 Build

The plan has ZERO mention of negative keywords. This is a critical oversight that will burn 20-40% of budget on irrelevant clicks. Build these negative lists before launching a single campaign.

### 2.1 Universal Negative Keyword List (Apply to ALL Campaigns)

**DIY/Free Seekers (filter for paid campaigns only):**
- free, cheap, discount, coupon, promo code, trial (use selectively -- free scan is the entry point, so only negate on paid conversion campaigns)

**Academic/Educational:**
- definition, meaning, essay, assignment, homework, school, university, student project, what is, wikipedia, scholarly

**Unrelated "Digital Footprint" Contexts:**
- carbon footprint, environmental footprint, ecological footprint, sustainability, carbon, CO2, green
- digital footprint lesson plan, teaching digital footprint, digital citizenship, kids, children, middle school, elementary

**Criminal/Legal:**
- criminal record, background check, arrest record, court records, mugshot, sex offender, FBI, police
- (Critical: "background check" queries will match broadly -- many are criminal background checks, not digital reputation)

**Technical/IT:**
- server footprint, data center footprint, hardware footprint, memory footprint, code footprint, software footprint, disk footprint

**Job-Adjacent but Wrong Intent:**
- salary, pay, compensation, benefits, remote jobs, job listings, job openings, hiring now, apply now
- (These are job-seekers looking for jobs, not career optimization)

### 2.2 Campaign-Specific Negatives

**Brand Campaign Negatives:**
- Competitor brand names (to avoid brand queries triggering competitor content)
- "review", "complaint", "scam", "legit" (route to separate reputation management campaign)

**Problem-Aware Campaign Negatives:**
- Brand terms (force these to brand campaign)
- Competitor names (force to competitor campaign)
- "tool", "software", "app", "service" (force to solution-aware campaign)

**Solution-Aware Campaign Negatives:**
- Pure informational modifiers: "what is", "definition", "meaning", "how does"
- Brand terms + competitor names (force to respective campaigns)

**Competitor Campaign Negatives:**
- Brand terms (avoid cannibalization)
- Non-competitor names that happen to match (e.g., "LinkedIn" could trigger broadly)

### 2.3 Negative Keyword Review Cadence
- **Daily** for first 2 weeks: review search terms report, add negatives
- **Twice weekly** for weeks 3-8
- **Weekly** ongoing after month 2
- Set up automated alerts for spend on any single query exceeding $50 without conversion

## 4. Cross-Campaign Overlap Risks

### 4.1 CRITICAL: Meta vs Google Keyword Overlap

The plan treats Meta (Phase 1) and Google (Phase 2) as independent channels with identical targeting concepts. This will cause:
- **Double attribution**: Same user sees Meta ad + searches on Google = both channels claim the conversion
- **Budget inflation**: Paying twice to acquire the same user
- **Recommendation**: Implement holdout tests. Run Google-only cohorts in some geos to measure true incrementality of each channel.

### 4.2 Problem-Aware vs Solution-Aware Campaign Overlap (Google)

The plan's tiers are:
- Tier 2 (Problem-aware, 40%): "digital footprint", "recruiters googling you"
- Tier 3 (Solution-aware, 30%): "career AI tools", "LinkedIn optimization", "online reputation"

**Overlap risk**: A query like "digital footprint tool" matches both. "Online reputation check" could match both. "How to fix digital footprint" is problem-adjacent but solution-intent.

**Fix**: Implement strict keyword match types:
- Problem-aware: Use phrase match and broad match modified only for pure problem queries
- Solution-aware: Use exact match and phrase match for tool/service/product queries
- Cross-negative both campaigns against each other's core terms

### 4.3 Brand vs Competitor Campaign Overlap

"Digital footprint scan" is both a brand term (Meldar's product name) AND a generic category term. If someone searches "digital footprint scan review", is that brand or competitor?

**Fix**:
- Brand campaign: Exact match [meldar], [meldar scan], [meldar digital footprint]
- Generic product campaign: Phrase match "digital footprint scan" with negative -meldar
- Competitor campaign: Exact match [competitor name] + phrase match "competitor alternative"

### 4.4 Programmatic SEO vs Google Ads Cannibalization

Section 3.2 plans 500 programmatic SEO pages for "Digital footprint for [job title]". Section 2.1 plans Google Ads on similar terms.

**Risk**: Paying for clicks on queries where you already rank organically.

**Fix**:
- Build an automated script that checks organic rankings weekly
- Add negatives for any query where organic position is 1-3
- Only run paid on queries where organic position is 4+ or absent
- This alone could save 10-15% of Google budget

### 4.5 Performance Max Cannibalization (Section 2.1)

The plan includes Performance Max campaigns. PMax is notorious for cannibalizing brand search traffic (cheap, high-converting clicks that would have come organically or via brand campaigns).

**Fix**:
- Add brand terms as negative keywords in PMax (Google now allows this)
- Monitor PMax search terms reports weekly
- Set PMax as lower priority than dedicated search campaigns
- Consider excluding PMax entirely until dedicated campaigns are mature (month 3+)

## 5. Query Sculpting Recommendations

### 5.1 Campaign Priority Structure (Google Ads)

Implement a tiered priority system to control which campaign serves which query:

```
Priority 1 (Highest): Brand Campaign
  - Match: Exact match on brand terms
  - Negatives: None needed (catches all brand queries first)
  - Bid strategy: Target Impression Share 95%+

Priority 2: Competitor Campaign
  - Match: Exact + phrase match on competitor names
  - Negatives: Brand terms
  - Bid strategy: Manual CPC (control costs on expensive competitor terms)

Priority 3: Solution-Aware Campaign
  - Match: Phrase + exact match on solution/tool/service queries
  - Negatives: Brand terms + competitor names + informational modifiers
  - Bid strategy: Target CPA

Priority 4: Problem-Aware Campaign
  - Match: Broad match + phrase match on problem queries
  - Negatives: Brand + competitor + solution terms + educational modifiers
  - Bid strategy: Maximize conversions with CPA cap
```

### 5.2 Match Type Architecture

**Current plan issue**: No match types specified. This is a major gap. If all keywords run on broad match, campaigns will overlap massively.

**Recommended approach:**
- **Brand**: Exact match only ([meldar], [meldar scan])
- **Competitor**: Exact match ([brandyourself], [reputation.com]) + phrase ("brandyourself alternative")
- **Solution-aware**: Phrase match ("digital footprint tool", "online reputation service") + exact match for top converters
- **Problem-aware**: Broad match with extensive negatives (let Google find long-tail queries, but control waste)

### 5.3 RLSA (Remarketing Lists for Search Ads) Integration

Not mentioned in the plan at all. Critical for query sculpting:
- Create RLSA audiences from: quiz starters, quiz completers, landing page visitors, paid users
- Bid +50-100% on problem-aware queries from users who already visited (they're further along the funnel than the query suggests)
- Run broad match campaigns ONLY for RLSA audiences (safe way to discover new queries without waste)

### 5.4 Query-to-Landing Page Mapping

The plan has a single primary landing page. Different query intents need different landing pages:

| Query Intent | Landing Page | CTA |
|---|---|---|
| "what do recruiters see" (problem) | Blog post with embedded quiz | "Check yours now" |
| "digital footprint tool" (solution) | Product comparison page | "Start free scan" |
| "meldar" (brand) | Main landing page | "Get your free scan" |
| "[competitor] alternative" (competitor) | Comparison page: Meldar vs X | "Try Meldar free" |
| "digital footprint for [job title]" (programmatic) | Job-title-specific page | "Scan for [title]" |
| "team digital footprint" (B2B) | Enterprise landing page | "Book a demo" |

## 7. Overall Assessment & Critical Recommendations

### Severity: HIGH -- The keyword strategy needs significant work before launch.

**Top 5 actions before spending a single dollar on Google Ads:**

1. **Build the negative keyword architecture** (Section 2 above). Without it, expect 25-40% wasted spend. The "carbon footprint" / "digital footprint" overlap alone will burn thousands.

2. **Implement match type controls and campaign priority sculpting** (Section 5). The current plan will have all campaigns competing against each other for the same queries.

3. **Add the "Google yourself" keyword cluster** (Section 6.1). This is the single highest-volume, highest-intent cluster for the product and it is completely absent from the plan.

4. **Rebalance budget allocation from problem-aware (40%) to solution-aware (40%)**. Problem-aware queries convert poorly on paid search; serve them with SEO/content instead. Solution-aware queries have 3-5x higher conversion rates.

5. **Build separate landing pages per intent** (Section 5.4). Sending all traffic to one landing page will tank Quality Scores and conversion rates. Google rewards relevance -- a "digital footprint tool" query landing on a page with that exact headline will score 7-8/10 QS vs. 4-5/10 on a generic page.

### Budget Impact Estimate

If the keyword strategy is implemented as currently written (no negatives, no match types, no query sculpting, single landing page):
- **Estimated waste**: 30-40% of Google Ads budget ($900-$1,200/mo at $3K spend)
- **Estimated CPA**: $12-20 (above kill threshold)

With the recommendations above implemented:
- **Estimated waste**: <8%
- **Estimated CPA**: $5-10
- **Estimated additional keyword volume captured**: +200-300% more eligible queries
