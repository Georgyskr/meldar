# Meldar

Your AI. Your app. Learn or fall behind

Meldar helps non-technical people discover what wastes their time, then builds personal apps to fix it.

- **Site:** [meldar.ai](https://meldar.ai)
- **Company:** ClickTheRoadFi Oy (Helsinki, Finland)
- **Stage:** Launched, building v3

## Repo layout

This is a pnpm + Turborepo monorepo.

```
apps/
  web/                Next.js 16 app (App Router, RSC, Panda CSS + Park UI)
packages/
  db/                 Drizzle schema + hand-written SQL migrations (Neon Postgres)
  orchestrator/       Build engine: events, state machine, streaming
  storage/            Provider abstraction (in-memory + postgres)
  sandbox/            Cloudflare Workers sandbox client
  tokens/             Design tokens shared across web + future native
  test-utils/         Shared Vitest helpers
  tooling-config/     Shared tsconfig + biome presets
```

Architecture, conventions, and engineering rules live in `CLAUDE.md` and `AGENTS.md` at the repo root.

## Tech stack

- **Framework:** Next.js 16 (App Router, React Server Components, React 19)
- **Styling:** Panda CSS + Park UI
- **Language:** TypeScript (strict)
- **Package manager:** pnpm 10
- **Build:** Turborepo
- **Lint + format:** Biome
- **Database:** Neon Postgres + Drizzle ORM
- **Auth:** custom JWT (see `apps/web/src/server/auth`)
- **AI:** Anthropic Claude (Haiku for OCR, Sonnet for orchestration)
- **Email:** Resend
- **Payments:** Stripe
- **Sandbox:** Cloudflare Workers + R2
- **Deployment:** Vercel
- **Analytics:** Google Analytics 4 (consent-gated)

## Common commands

From the repo root:

```bash
pnpm install                 # install all workspaces
pnpm dev                     # turbo run dev (starts apps/web on :3000)
pnpm build                   # production build (all workspaces)
pnpm typecheck               # tsc --noEmit across all workspaces
pnpm test                    # vitest run across all workspaces
pnpm format-and-lint         # biome check .
pnpm format-and-lint:fix     # biome check --fix .
```

Per-app:

```bash
pnpm --filter @meldar/web dev
pnpm --filter @meldar/web build
pnpm --filter @meldar/web test
```

## Database

Migrations live in `packages/db/migrations/` as hand-written, idempotent SQL files. They are **not** generated by Drizzle Kit because v3 needs features Drizzle Kit can't emit (`CREATE EXTENSION`, `DEFERRABLE` foreign keys, partial indexes with complex predicates).

Apply order, per environment (dev → staging → prod):

```bash
psql "$DATABASE_URL" -f packages/db/migrations/0000_meldar_v3_initial.sql
psql "$DATABASE_URL" -f packages/db/migrations/0001_projects_preview_url.sql
```

Setup runbook with the full Neon configuration (compute size, role-level timeouts, `pg_stat_statements`, hot-path monitoring) is at `docs/v3/engineering/sprint-1-db-setup.md`.
